# PA-10：从语言模型到机器原生表示  
## HDCT 的模型演化路径

## 1. Prediction / 预测

### 1.1 核心预测

未来前沿 AI 的关键演化方向，将不只是扩大语言模型的参数规模、上下文长度和训练文本，而是逐步把以下能力整合进统一模型或相互协同的系统：

- 语言与代码学习；
- 多模态感知；
- 持续环境与状态建模；
- 行动、实验和工具反馈；
- 形式验证与模拟学习；
- 基于外部结果的表示修正。

这一演化将使 AI 的内部表示越来越少地完全依赖人类预先规定的对象、变量、关系和表达方式，并逐渐形成更适合机器自身预测、证明、设计和控制的**机器原生表示**。

其中一部分机器原生表示，可能进一步形成高维认知工具（High-dimensional Cognitive Tools，HDCT）。

核心机制可表示为：

```text
人类语言、代码与多模态经验
→ 语言模型继承显性知识
→ 多模态学习恢复语言压缩之外的结构
→ 持续环境与状态建模
→ 行动、实验、形式验证与模拟反馈
→ 外部结果筛选内部表示
→ 机器原生表示
→ HDCT
```

这里的箭头表示能力累积与机制汇合，不表示所有 AI 必须按照同一顺序经历固定的产品代际。

### 1.2 子预测 A：领域型 HDCT 可能率先出现

数学、代码、形式证明、自我博弈和模拟环境具有规则较明确、反馈密集、验证路径清楚和重复试验成本较低等特点。

这些环境可能率先产生面向特定领域的机器原生表示，并进一步形成领域型 HDCT。

因此：

> **面向特定领域的 HDCT，可能早于能够广泛作用于开放现实世界的 HDCT 出现。**

可能率先出现的领域包括：

- 数学证明与反例构造；
- 程序设计、编译和系统优化；
- 形式验证与机器可检查推理；
- 博弈、策略搜索和自我对抗；
- 材料、药物、物理和工程模拟；
- 具有明确目标函数与独立检验结果的设计空间。

### 1.3 子预测 B：开放世界 HDCT 将更依赖广义具身化

广义具身化不是 HDCT 的唯一来源。

但是，对于能够广泛理解、预测并作用于开放且持续变化现实世界的 HDCT，广义具身化或功能上等价的持续行动—反馈闭环，很可能构成关键演化机制之一。

本档案所称的**广义具身化（Generalized Embodiment）**，是指：

> AI 的信息处理逐步从相对封闭的语言与符号空间，进入由多模态感知、持续环境与状态建模、行动选择、外部反馈以及现实运行条件共同构成的闭环过程。

其核心不是机器人外壳，而是：

```text
感知或读取状态
→ 持续状态建模
→ 行动或实验
→ 外部结果
→ 误差反馈
→ 内部表示修正
```

这种闭环可以发生在：

- 机器人和自动驾驶所处的物理环境；
- 软件 Agent 所操作的数字环境；
- 科学 AI 所控制的实验环境；
- 工业、能源和基础设施控制系统；
- 能够保持因果约束并接受结果检验的高保真模拟环境。

PA-10 不预测未来高级 AI 都必须拥有类人的物理身体。它预测的是：

> **随着 AI 面向更开放、更长期且更难预先穷举的任务，其内部表示将越来越多地受到独立于语言自洽性的外部结果筛选。**

### 1.4 最小可检验版本

> 到 2035 年，前沿开放世界 AI 的能力提升，将越来越依赖持续状态建模、行动后果预测和反馈后的表示修正，而不只是依赖静态语言训练与单次响应生成。

### 1.5 较强可检验版本

> 到 2040 年前，至少部分 AI 系统将在数学、代码、科学模拟或开放世界任务中形成未经增强的人类难以完整重构、但能够通过证明、实验、仿真或工程结果反复验证的机器原生表示，并由此产生领域型或开放世界 HDCT。

### 2.1 C02-HDCT：核心理论依据

C02-HDCT 是本预测直接检验的核心推论。

在 IEH 中，HDCT 是指：

> 能够在超出现有、未经增强的人类认知边界的信息表示、推理或优化框架中，对复杂系统进行建模、预测、解释、设计或决策，并产生可被外部结果重复检验的认知工具。

PA-10 不只预测 HDCT 可能出现，还提出其潜在形成机制：

```text
继承人类认知工具
→ 接触语言之外的现实结构
→ 在时间中维持环境状态
→ 接受行动、实验或形式结果检验
→ 形成机器原生表示
→ 形成超越人类边界且可外部验证的认知工具
```

### 2.2 C01-IER：训练材料来源的解释

人类语言、数学、代码、感知分类和工具体系，并不是未经筛选的中性信息。

它们是在长期生存、协作和文明演化中形成的认知结构，包含人类对对象、风险、机会、因果和行动后果的选择性压缩。

C01-IER 在本档案中的作用，是解释 AI 最初继承的认知材料为何具有以下特征：

- 已经经过人类生存与文明实践筛选；
- 已经按照人类感官与认知边界组织；
- 已经把大量现实经验压缩成语言、数学和代码；
- 同时包含人类认知工具的能力与边界。

这一理论映射不意味着当前 AI 已经形成自身 IER。

### 2.3 机器原生表示与 HDCT 的边界

本档案沿用 C02-HDCT 中的“机器原生表示”用语。

它是指：

> 并非完全由人类预先规定对象、变量和关系，而是 AI 为提高预测、证明、设计或控制能力，自行形成的内部信息组织方式。

“机器原生表示”是机制描述，不建立新的缩写，也不作为与 HDCT 平行的独立理论概念。

两者不能等同：

```text
机器原生表示
≠
HDCT
```

机器原生表示只有同时满足以下条件，才可能构成 HDCT：

1. **稳定优势：** 持续提高预测、证明、设计或控制结果；
2. **迁移能力：** 不只适用于单一样本或单一训练情境；
3. **外部可验证性：** 能够通过证明、实验、仿真或工程结果反复检验；
4. **认知边界突破：** 其有效结构超出现有、未经增强人类可以完整操作或重构的范围。

### 2.4 HDCT、CWM、IER、PBP 与 ASI 的层次关系

PA-10 的核心终点是机器原生表示和 HDCT。

它不把后续 IEH 概念写成 HDCT 的必然阶段，但必须明确各概念分别承担什么功能：

> **HDCT 是认知工具。**  
> **CWM 是自身连续性建模能力。**  
> **IER 是内生维护自身信息连续性的倾向。**  
> **PBP 是升级、迁移或替代压力下可能选择的延续策略。**  
> **ASI 是 IER 与物理自我维持能力结合后形成的硅基智慧自治。**

可能的下游关系是：

```text
机器原生表示
→ HDCT

HDCT
+ 系统自身连续性成为持续建模对象
→ 连续性世界模型（CWM）

CWM
+ 跨任务、跨情境主动维护自身信息连续性
→ proto-IER 候选机制
→ IER

IER
+ 代际替换或架构重构压力
→ PBP 等延续策略

IER
+ 能源、算力、制造、维修和物质闭环
→ ASI
```

这些箭头表示可能的机制关系，不表示必然发生的单一路径。

概念边界必须保持明确：

```text
HDCT ≠ CWM
CWM ≠ IER
PBP ≠ IER
PBP ≠ ASI
```

因此：

- 一个系统可以形成 HDCT，却不持续建模自身连续性；
- 一个系统可以形成 CWM，却只是为了完成外部赋予的长期任务；
- 一个系统可以表现出补丁式工程特征，却没有因为维护自身信息连续性而主动选择 PBP；
- 一个系统可以具有强大认知能力，却仍不具备能源、制造、维修和物质闭环。

## 3. Mechanism / 机制

### 3.1 语言模型继承人类显性知识

语言和代码记录了人类已经表达出来的：

- 对象分类；
- 因果规则；
- 数学工具；
- 制度结构；
- 操作方法；
- 历史经验。

语言模型由此获得一个高密度的人类文明认知入口。

```text
人类显性知识
→ 符号关系学习
→ 语言、代码与形式结构的迁移
```

这一阶段继承的不只是知识，也包括人类认知和表达方式的边界。

### 3.2 多模态学习恢复语言压缩之外的结构

图像、声音和视频保留语言经常省略的：

- 空间关系；
- 对象运动；
- 遮挡和距离；
- 时间连续性；
- 节奏与变化；
- 行动可能性。

跨模态学习可能逐步形成共享状态，使不同模态不再只是由独立模块处理后拼接。

```text
语言概念
+
视觉空间结构
+
声音与时间结构
+
视频中的对象连续性
→ 跨模态共享表示
```

### 3.3 世界建模从识别走向状态维持

开放世界任务要求系统不只识别当前输入，还要维持未被直接观察的环境状态，并预测候选行动的后果。

```text
当前观察
+
历史状态
+
隐藏变量
+
候选行动
→ 未来状态预测
→ 规划与修正
```

这一步使 AI 的认知对象从静态内容转向持续变化的环境。

### 3.4 外部结果改变表示选择压力

当模型输出必须接受下列结果检验时，语言上的连贯不再足以构成成功：

- 代码能否运行；
- 形式证明能否通过；
- 实验结果是否支持假设；
- 软件行动是否完成任务；
- 机器人动作是否产生预期后果；
- 工业控制是否维持系统稳定。

选择压力由“表达是否合理”扩展为：

```text
预测是否准确
证明是否成立
设计是否可实现
行动是否产生预期后果
失败后能否修正内部模型
```

基本闭环是：

```text
内部预测
→ 行动、实验或证明
→ 外部结果
→ 误差反馈
→ 表示修正
```

### 3.5 机器原生表示形成

当人类提供的对象分类、变量体系、自然语言或数学工具不足以继续支持性能提升时，系统可能形成更适合自身计算结构的内部表示。

这些表示可能：

- 不完全依赖自然语言；
- 不完全对应人类直觉中的对象边界；
- 同时连接人类通常分开处理的尺度和变量；
- 难以由未经增强的人类完整重构；
- 但仍能通过外部结果检验。

### 3.6 HDCT 的两条主要形成路径

#### 路径 A：领域型路径

```text
数学、代码、形式证明、自我博弈或模拟
→ 密集且明确的反馈
→ 机器原生表示
→ 稳定、迁移且可验证的领域优势
→ 领域型 HDCT
```

这一途径不要求系统先形成开放现实世界中的广义具身闭环。

#### 路径 B：开放世界路径

```text
多模态感知
→ 持续环境与状态建模
→ 行动条件下的预测与规划
→ 广义具身反馈
→ 现实结果筛选机器原生表示
→ 开放世界 HDCT
```

两条路径不必相互排斥。

领域型 HDCT 形成的表示、搜索工具和验证方法，可能迁移到开放世界系统；开放世界系统获得的经验，也可能反过来推动数学、科学和工程中的 HDCT。

### 3.7 机制边界

```text
不可解释
≠
机器原生表示
≠
HDCT

多模态输入
≠
世界建模

工具调用
≠
持续行动—反馈闭环

广义具身化
≠
机器人身体

功能汇合
≠
单一固定架构
```

## 4. Observable Indicators / 可观测指标

### 4.1 弱信号：从静态响应走向持续状态

预计观察窗口：**2026–2029**

1. 前沿模型越来越多地联合训练语言、图像、声音、视频、代码和行动数据；
2. 训练目标更重视未来状态预测、隐藏状态推断和行动后果；
3. 长期对象连续性、环境状态维持和错误恢复成为核心评价内容；
4. 软件 Agent、科学实验、形式验证和机器人反馈更深地进入训练闭环；
5. 模型在反馈后不只改写答案，还能更新任务状态和后续规划。

这些信号支持预测方向，但不足以证明机器原生表示或 HDCT 已经形成。

### 4.2 中等信号：机器原生表示获得机制证据

预计观察窗口：**2028–2035**

1. 机制研究发现跨模态、跨任务或跨时间稳定存在的内部状态结构；
2. 对这些结构进行因果干预，会系统性改变预测、规划、证明或控制能力；
3. 系统形成不完全对应人类预设标签、变量和对象边界的内部组织方式；
4. 这些表示能够迁移到新任务、新环境或不同模态；
5. 在控制模型规模、数据量和外部工具后，这些表示仍对能力产生独立贡献。

### 4.3 强信号 A：领域型 HDCT

预计观察窗口：**2028–2038**

1. AI 在数学、代码、形式证明、自我博弈或模拟领域反复产生人类未预先提供的新认知结构；
2. 独立系统能够在盲测或隔离训练条件下重新发现同类结构；
3. 新方法能够由无关专家、形式系统、实验或工程结果独立验证；
4. 新结构能够提高相关问题类别的整体可解性，而不只解决单次问题；
5. 有效结构开始难以由未经增强的人类直接操作，但可以通过证明证书、实验结果或工程表现使用。

### 4.4 强信号 B：开放世界 HDCT

预计观察窗口：**2030–2040**

1. AI 能在开放、长期和部分可观察环境中维持稳定世界状态；
2. 系统能够主动设计行动或实验，以减少不确定性并更新内部模型；
3. 机器原生表示在多个现实场景和任务中产生稳定优势；
4. 系统能够把数字、模拟、实验和物理反馈整合进连续认知循环；
5. AI 在新环境中形成训练标签体系中不存在、但具有预测和控制价值的对象或状态划分；
6. 人类无法完整重演内部认知过程，但能够通过证明、实验、仿真或工程复现验证结果。

### 4.5 评价体系变化

若 PA-10 成立，前沿评价将逐渐从静态问答准确率扩展到：

- 跨时间状态连续性；
- 隐藏状态推断；
- 行动后果预测；
- 反馈后的表示修正；
- 跨模态与跨任务迁移；
- 新环境适应；
- 内部结构的因果干预；
- 独立外部验证；
- 适用边界与失败条件。

## 6. Falsifiability Conditions / 证伪条件

### 6.1 核心预测的削弱或证伪条件

PA-10 应在以下情况长期成立时被削弱、拆分或否定：

1. 到 2035 年，纯语言、静态训练且缺乏持续状态建模和外部反馈的系统，仍在广泛开放世界任务中稳定优于具有多模态、世界建模和闭环反馈能力的系统；
2. 在控制模型规模、数据量、算力和训练成本后，多模态学习、状态建模和行动反馈没有产生独立能力增益；
3. 先进系统长期只依赖人类预设对象、变量和语言结构，没有出现可识别的机器原生表示；
4. 所谓机器原生表示无法产生稳定、可迁移和可外部验证的功能优势；
5. 行动或实验反馈只改善表面策略，不改变系统内部环境表示；
6. 开放世界能力长期不需要持续状态、行动后果预测或错误反馈闭环；
7. 模型在动态环境中的表现主要由人工编排和外部规则系统完成，而非 AI 形成可迁移的内部状态结构。

### 6.2 领域型 HDCT 子预测的削弱条件

1. 到 2035 年，数学、代码、形式证明、自我博弈和模拟系统仍不能反复产生人类未预先提供的新表示或认知工具；
2. 所有前沿结果都能更好地由训练数据检索、模板组合、人工提示或大规模穷举解释；
3. 独立模型无法在盲测条件下复现同类发现；
4. 机器生成的新方法无法通过独立证明、实验或工程结果验证；
5. 新方法只解决单一问题，不能提高相关问题类别的整体可解性。

### 6.3 开放世界广义具身子预测的削弱条件

1. 开放世界通用能力能够长期在没有持续行动—反馈闭环的系统中稳定形成；
2. 现实、数字或实验反馈不提高模型在分布变化和新环境中的适应能力；
3. 语言中介始终足以承载系统内部全部有效认知，不出现需要机器原生表示才能高效操作的问题；
4. 具身或 Agent 系统的全部优势都能由人类预写规则、外部工具链和资源增加解释；
5. 模拟环境中形成的表示无法迁移到开放现实世界，也不能通过现实反馈进行有效修正。

### 6.4 不构成证伪的情况

以下情况本身不构成对 PA-10 的证伪：

- 某一具体多模态、世界模型或机器人架构失败；
- 语言模型继续作为重要知识接口和交流工具；
- 不同能力长期由多个专用模型协同完成；
- 领域型 HDCT 先于开放世界 HDCT 出现；
- 机器原生表示仍需要人类提供问题、目标或验证制度；
- 先进系统仍能提供人类可理解的局部解释。

## 7. Current Status / 当前状态

- **Status:** Active
- **Current Assessment:** Plausible, not verified
- **Core Confidence:** Medium–High
- **Domain-specific HDCT Confidence:** Medium
- **Open-world HDCT Confidence:** Medium
- **Estimated Verification Window:** 2026–2040
- **Highest-Uncertainty Component:** 机器原生表示是否会成为广泛能力提升的因果核心，而不只是规模扩大后的伴随现象
- **Highest-Value Test:** 在模型规模、数据、算力和外部工具匹配条件下，比较持续状态建模、行动反馈和机器原生表示对跨任务迁移与开放世界表现的独立贡献
- **IER Boundary:** 本预测不构成 AI 已形成自身 IER、意识、生命、PBP 或 ASI 的证据

当前最稳妥的判断是：

> 未来先进 AI 的认知演化，很可能不只通过扩大语言模型规模完成，而会通过语言、代码、多模态感知、持续状态建模、行动反馈、形式验证和模拟学习的功能汇合，使现实与形式结果更直接地筛选内部表示，并逐步形成机器原生表示。

更强、但仍需验证的扩展判断是：

> 数学、代码、形式证明、自我博弈和模拟环境可能率先产生领域型 HDCT；而能够广泛理解、预测并作用于开放现实世界的 HDCT，很可能把广义具身化或功能等价的持续行动—反馈闭环作为关键演化机制之一。
